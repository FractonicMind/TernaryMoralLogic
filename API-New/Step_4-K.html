Introduction

The TML Specification Architecture v3.3 establishes the canonical interface between moral governance and computational execution through two interdependent deliverables: tml_abi.json defines the on-chain constitutional enforcement interface, while eip712_typed_data.json enables off-chain structured data hashing with on-chain verification. Together, they operationalize the triadic logic core, enforce the binary-ternary parallel architecture, and implement the "No Log = No Action" iron law through strict schema constraints.

This specification package serves as the cryptographic bridge between binary inference engines that propose actions with speed and pattern recognition, and ternary governance coprocessors that dictate whether those actions physically cross the execution threshold. The dual-lane latency architecture—Inference Lane (<2ms) and Anchoring Lane (<500ms)—ensures that hardware governance does not destroy throughput, neutralizing the fatal industry objection to moral computation.

Every endpoint and type definition in this specification is annotated with x-tml-pillar identifiers mapping to the Eight Pillars, and x-tml-monograph-ref citations to specific monograph sections. Implementation status is explicitly declared per the decision matrix: SHIPPING for buildable capabilities, BETA for documented tradeoffs, and FUTURE for blocked features with explicit constraint citations.

Type Definitions and Encoding Rules

Atomic Type Mappings

EIP-712 Type JSON Schema Type Encoding Notes
int8 integer (int8) Signed, restricted to {-1, 0, +1}
uint256 string (decimal) Big integer, may exceed JS safe range
bytes32 string (hex) Fixed length, 0x-prefixed
address string (hex) 20 bytes, padded to 32 in ABI

Hash Struct Computation

1. typeHash = keccak256(encodeType(primaryType))
2. For each field: encode according to type rules
3. Concatenate typeHash ‖ encodedData
4. Return keccak256 of concatenation

Final Signing Hash:
keccak256("\x19\x01" ‖ domainSeparator ‖ hashStruct(message))
